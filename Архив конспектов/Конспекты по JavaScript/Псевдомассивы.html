Псевдомассивы

Метод querySelectorAll возвращает коллекцию NodeList. Она похожа на обычный массив: элементы упорядочены, можно обратиться к свойству length или вызвать forEach. Но остальные методы массива не сработают: map, reduce, filter и sort.

Если попытаться их применить, вылезет ошибка:

const posts = document.querySelectorAll('.post');

const updatedPosts = posts.map((post) => {
  post.textContent = 'Никаких больше вечеринок';
});

// TypeError: posts.map is not a function
// потому что у posts нет метода map

querySelectorAll — не единственное место, где мы можем столкнуться с ненастоящим массивом. Свойство children хранит дочерние элементы в виде коллекции под названием HTMLCollection.

У неё вообще нет методов, даже forEach:

const container = document.querySelector('.posts-container');

container.children.forEach((item) => {
  console.log(item);
});

// TypeError: container.children.forEach is not a function

Скорее всего, любая похожая на массив коллекция, с которой вы столкнётесь при работе с DOM, будет псевдомассивом. Возникает вопрос — можно ли как-то сделать из такой коллекции полноценный массив?

Можно — на помощь приходит ES2015.

Есть быстрый способ превратить любой псевдомассив в массив. Для этого в ES2015 появился метод Array.from. Он создаёт массив из «массивоподобного» объекта:

const posts = content.querySelectorAll('.post');

const postsArray = Array.from(posts); // такой вызов вернёт полноценный массив

Объект считается массивоподобным, если:

  • его элементы имеют числовые индексы,
  • у него есть свойство length.

Метод querySelectorAll вернёт коллекцию, которая под эти критерии подходит. Поэтому такой код сработает:

const posts = document.querySelectorAll('.post');

const updatedPosts = Array.from(posts).map((post) => {
  post.textContent = 'Никаких больше вечеринок';
});
